## 三、修复详情

### Bug B1 — l1_dcache.v：IDLE命中判定使用滞后地址

**问题根因：**  
原代码在 `IDLE` 状态检查 `hit` 信号时使用寄存器 `pend_addr`，该寄存器在同一周期才被赋值（非阻塞赋值），因此命中逻辑实际比较的是上一拍地址。

**修复方案：**  
```verilog
// 修复前（错误）：使用滞后寄存器
assign hit = (pend_addr == tag_addr) && valid[way];

// 修复后（正确）：直接使用组合输入
wire [TAG_W-1:0] req_tag  = cpu_addr[ADDR_W-1 : INDEX_W+OFFSET_W];
assign hit = (tag_arr[set][way] == req_tag) && valid[set][way];
```

同时将 `cpu_rdata` 从 `reg` 改为 `wire`，实现组合输出：
```verilog
// 修复前
reg  [31:0] cpu_rdata;
always @(posedge clk) cpu_rdata <= data_arr[hit_way][word_off];

// 修复后
wire [31:0] cpu_rdata = data_arr[hit_set][hit_way][word_off];
```

**效果：** 消除1拍读命中延迟，性能对齐报告书IPC分析。

---

### Bug B2 — l2_cache.v：RD_R 状态 NBA 竞争 + 缺少NC通路

**问题根因：**  
在 `RD_R`（填充完成）状态，同一 `always` 块通过非阻塞赋值（NBA）写入 `data_arr`，同一周期另一路径读出 `data_arr` 送给上游。Verilog NBA 语义在同周期内写入不可见，导致返回的数据为旧值（垃圾数据或全0）。

**修复方案：**  
新增 `FILL_DONE` 状态，将写入与读出分开两拍：
```
... → FILL → RD_R（写入data_arr）→ FILL_DONE（读出并返回）→ IDLE
```

同时新增 NC passthrough 状态机分支：
```
IDLE --[NC地址]--> NC_RD_REQ → NC_RD_WAIT → NC_RD_RESP → IDLE
     --[NC地址]--> NC_WR_REQ → NC_WR_WAIT → IDLE
```

---

### Bug B3 — dma_regfile.v：AXI4 AW+W 须同周期（违反规范）

**问题根因：**  
原代码写地址通道（AW）与写数据通道（W）握手逻辑如下：
```verilog
// 错误：要求两通道同周期valid
if (s_awvalid && s_wvalid) begin
    addr_latch <= s_awaddr;
    data_latch <= s_wdata;
end
```
AXI4规范明确允许W先于AW到达，或AW先于W到达。

**修复方案：**  
引入独立锁存寄存器：
```verilog
// 独立握手
always @(posedge clk) begin
    if (s_awvalid && s_awready) aw_latch <= s_awaddr;
    if (s_wvalid  && s_wready ) w_latch  <= s_wdata;
end
// 两者都锁存后执行写操作
assign do_write = aw_done && w_done;
```

### Bug B4 — soc_top.v：s1_bid[4]/s1_rid[4] 悬空

**问题根因：**  
AXI4 BID/RID 宽度为5位（支持最多32个事务ID）。DMA cfg从机（slave 1）的 `s1_bid` 与 `s1_rid` 的第4位（最高位）未连接任何驱动，仿真产生X传播，综合产生未连接警告。

**修复方案：**  
所有DMA cfg事务均来自Master 0（L2 Cache），其ID范围为 `5'b0xxxx`，最高位恒为0，因此直接拉低：
```verilog
assign s1_bid = {1'b0, s1_bid_inner[3:0]};
assign s1_rid = {1'b0, s1_rid_inner[3:0]};
```
